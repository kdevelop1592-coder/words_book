# 관리자 페이지 구현 계획서

이 문서는 `words_book` 프로젝트에 관리자 전용 대시보드를 추가하기 위한 기술적 구현 계획을 담고 있습니다.

## 🎯 목표
- 가입한 모든 사용자의 목록 및 등록한 단어 수 확인
- 사용자별 퀴즈 연습 상황 및 오답률 점검
- 관리자 전용 UI 제공

## 🛠 주요 변경 사항

### 1. 백엔드 (src/firebase.js)
- **사용자 자동 등록**: 로그인 시 `users` 컬렉션에 사용자 정보(UID, 이름, 이메일)를 `setDoc(merge: true)`로 자동 저장/업데이트합니다.
- **퀴즈 결과 저장**: 퀴즈 종료 시 점수, 전체 문제 수, 오답 리스트를 `quiz_results` 컬렉션에 저장하는 `saveQuizResult` 함수를 추가합니다.
- **관리자 권한 확인**: `kdevelop1592@gmail.com` 계정을 관리자로 식별하는 `isAdmin` 함수를 구현합니다.
- **관리자 전용 데이터 조회**: 전체 사용자 통계 및 퀴즈 요약 데이터를 가져오는 `getAdminSummaries` 함수를 추가합니다.

### 2. 프론트엔드 HTML (index.html)
- **관리자 탭 추가**: 네비게이션 바에 `관리자` 버튼을 추가합니다 (기본값: `hidden`).
- **관리자 세션 추가**: 
    - 사용자 통계 테이블 (이름, 이메일, 단어 수)
    - 퀴즈 성적 요약 (평균 점수, 가장 많이 틀린 단어 TOP 10)

### 3. 프론트엔드 스타일 (style.css)
- 관리자 페이지용 글래스모피즘 테이블 디자인 및 통계 카드 스타일을 추가합니다.

### 4. 프론트엔드 로직 (src/main.js & src/quiz.js)
- **권한 기반 UI**: 로그인한 사용자가 관리자인 경우에만 관리자 탭을 노출합니다.
- **오답 추적**: `WordQuiz` 클래스에서 틀린 단어를 기록하도록 수정합니다.
- **대시보드 렌더링**: 관리자 탭 클릭 시 Firestore에서 데이터를 집계하여 화면에 출력합니다.

## 🧪 검증 계획
1. **사용자 등록**: 새 계정 로그인 시 Firestore `users`에 데이터가 생성되는지 확인합니다.
2. **퀴즈 결과**: 퀴즈 종료 후 `quiz_results`에 오답 데이터가 정확히 저장되는지 확인합니다.
3. **관리자 접속**: `kdevelop1592@gmail.com`으로 로그인 시 관리자 메뉴가 나타나는지 확인합니다.
4. **데이터 집계**: 관리자 페이지에서 전체 사용자의 학습 현황이 올바르게 요약되어 표시되는지 확인합니다.
